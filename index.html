    Authentication

    Create Account

    Create a new user with email and password. An email will be sent with a confirmation token.

    HTTP Request

    POST /api/user/signup

    
{
    "user" : {
        "email": "test@test.com",
        "password": "p4ssw0rd"      
    }
}

    HTTP Response

    {
    "user" : 
        {
            "uuid":"99e09614-d1d6-418c-b3de-239e44a1c220",
            "email":"test@test.com",
            "first_name":null,
            "last_name":null,
            "api_token":".......",
            "avatar_url":null
        }
}

    Confirm Account

    After sign up an email will be sent containing a confirmation token. This endpoint is to submit that token.

    HTTP Request

    POST /api/user/confirm

    
{
    "user" : {
        "uuid": "",
        "confirmation_token": "123456"      
    }
}

    HTTP Response

    
{
    "user" : 
        {
            "uuid":"99e09614-d1d6-418c-b3de-239e44a1c220",
            "email":"test@test.com",
            "first_name":null,
            "last_name":null,
            "api_token":".......",
            "avatar_url":null
        }
}

    Login (Email/Pass)

    Login using email and password. After user has confirmed. api_token in the response will be used for subsequent api requests.

    HTTP Request

    POST /api/user/login

    
{
    "user" : {
        "email": "",
        "password": "123456"        
    }
}

    HTTP Response

    
{
    "user" : 
        {
            "uuid":"99e09614-d1d6-418c-b3de-239e44a1c220",
            "email":"test@test.com",
            "first_name":null,
            "last_name":null,
            "api_token":".......",
            "avatar_url":null
        }
}

    Login (Providers: FB, G+)

    Login and/or account creation for users accessing via a 3rd party authentication (oAuth2). If logging in for the first time, an account will be created for the user using their provider user id, first name, last name obtained from the 3rd party.

    api_token in the response will be used for subsequent api requests.

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`’s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
